You’ve essentially got a client-side React component (.jsx), not a full app. To run this on Kubernetes, you need to:

1. Wrap it in a React app (Vite/CRA/Next etc.)


2. Build static assets


3. Serve them via a web server (e.g., NGINX)


4. Containerize


5. Deploy to Kubernetes



I’ll give you a production-grade, minimal path (Vite + NGINX + K8s).


---

1. Create React App Structure

Use Vite (fast, minimal):

npm create vite@latest eks-cost-ui -- --template react
cd eks-cost-ui
npm install

Replace src/App.jsx with your provided JSX code.

Update main.jsx (default is fine):

import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)


---

2. Build the App

npm run build

This generates:

dist/
  index.html
  assets/


---

3. Dockerfile (Production-Ready)

Use multi-stage build (Node → NGINX):

# ---- Build Stage ----
FROM node:20-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

# ---- Runtime Stage ----
FROM nginx:1.27-alpine

# Remove default nginx config
RUN rm -rf /usr/share/nginx/html/*

# Copy built app
COPY --from=builder /app/dist /usr/share/nginx/html

# Optional: custom nginx config for SPA routing
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]


---

nginx.conf (Important for React routing)

server {
  listen 80;

  root /usr/share/nginx/html;
  index index.html;

  location / {
    try_files $uri /index.html;
  }

  error_page 500 502 503 504 /50x.html;
}

4. Build & Push Image

Replace with your registry:

docker build -t your-dockerhub-username/eks-cost-ui:1.0 .
docker push your-dockerhub-username/eks-cost-ui:1.0


---

5. Kubernetes Manifests

Namespace (optional)

apiVersion: v1
kind: Namespace
metadata:
  name: eks-cost


---

Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: eks-cost-ui
  namespace: eks-cost
spec:
  replicas: 2
  selector:
    matchLabels:
      app: eks-cost-ui
  template:
    metadata:
      labels:
        app: eks-cost-ui
    spec:
      containers:
        - name: eks-cost-ui
          image: your-dockerhub-username/eks-cost-ui:1.0
          ports:
            - containerPort: 80
          resources:
            requests:
              cpu: "100m"
              memory: "128Mi"
            limits:
              cpu: "300m"
              memory: "256Mi"


---

Service

apiVersion: v1
kind: Service
metadata:
  name: eks-cost-ui
  namespace: eks-cost
spec:
  type: ClusterIP
  selector:
    app: eks-cost-ui
  ports:
    - port: 80
      targetPort: 80


---

Ingress (for external access)

If using NGINX Ingress Controller:

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: eks-cost-ui
  namespace: eks-cost
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
    - host: eks-cost.yourdomain.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: eks-cost-ui
                port:
                  number: 80

8. Key Observations (Important for your app)

1. This is a frontend-only app

No backend required

Calls external API (Anthropic) directly from browser


2. Security Concern ⚠

You're exposing API tokens in frontend:

"x-api-key": token

Better approach:

Add a backend proxy (Node/Go)

Store API key server-side

Frontend calls your API instead


3. CORS Consideration

Ensure your API endpoint supports browser requests.
