helm-managed rbac
Platform Admin
ClusterRole · Helm: platform-rbac
  • All API groups (*) full CRUD
  • All namespaces incl. platform NSes
  • Node, PV, ClusterRole management
  • CRD create/update/patch/delete
  • Karpenter NodeClass & NodePool admin
  • ArgoCD Application admin
  • Secrets, ConfigMaps full access
Group: platform-admins
Platform ReadOnly
ClusterRole · Helm: platform-rbac
  • get, list, watch on all resources
  • All namespaces incl. platform NSes
  • Nodes, PVs, PVCs read
  • Events, Logs read access
  • Karpenter resources read
  • ArgoCD Applications read
  • Secrets LIST only (no get/watch)
Group: platform-readers
Tenant Write
Role (NS-scoped) · Helm: tenant-rbac
  • Full CRUD on own namespace resources
  • Deployments, Services, Ingress
  • ConfigMaps, Secrets (own NS)
  • HPA, PodDisruptionBudget
  • ServiceAccounts (own NS only)
  • Pod exec, logs, port-forward
  • ❌ No platform NS access
Group: tenant-{name}-writers
Tenant ReadOnly
ClusterRole · Helm: platform-rbac
  • get, list, watch cluster-wide
  • Pods, Deployments, Services
  • Nodes read (for capacity view)
  • Events read cluster-wide
  • ❌ No Secrets get/watch
  • ❌ No platform NS deep access
  • ❌ No exec or port-forward
Group: tenant-readers

🔄 Tenant Onboarding Flow
1. Tenant opens PR to platform-gitops
2. Platform reviews and merges
3. ArgoCD creates tenant namespace
4. Terraform provisions IAM role + Access Entry
5. Helm chart installs Role + RoleBinding
6. Tenant gets write to tenant-gitops repo
7. Tenant self-serves Karpenter + workloads
🛡️ Security Controls
▸ ArgoCD RBAC limits tenant projects
▸ Tenants cannot deploy to platform NS
▸ NetworkPolicy isolates tenant traffic
▸ Resource quotas enforced per NS
▸ OPA/Gatekeeper policy enforcement
▸ Audit logs via CloudTrail + k8s audit
▸ Secrets via External Secrets + ASM
